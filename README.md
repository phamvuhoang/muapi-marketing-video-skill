# MuAPI Video Marketing for Claude

Create AI-powered marketing videos directly from **Claude Code CLI** or **Claude Desktop** using [MuAPI](https://muapi.ai)'s generative media APIs.

```
Brief → Storyboard → Images → Videos → Music → Delivery
```

---

## What You Can Build

| Request | Pipeline |
|---------|----------|
| "Create a 30s TikTok for our coffee brand" | Storyboard → images → clips → music |
| "Make a product reveal video" | Image upload → image-to-video → effects → music |
| "Animate this logo into a video ad" | Upload → image-to-video with zoom → music |
| "Add background music to my video" | Suno music gen → deliver audio URL |
| "Apply cinematic film noir effect to my clip" | Video effects API |
| "Lipsync this spokesperson to my voiceover" | Lipsync API |

## Full Setup Guide (New PC)

Follow these steps on a fresh machine to go from zero to generating videos with Claude.

### Prerequisites

Install the following before starting:

**Node.js v18+**
```bash
# macOS (via Homebrew)
brew install node

# Windows — download installer from https://nodejs.org
# Linux (Ubuntu/Debian)
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

Verify:
```bash
node -v   # should print v18.x.x or higher
npm -v    # should print 9.x.x or higher
```

**Git**
```bash
# macOS
brew install git

# Windows — download from https://git-scm.com
# Linux
sudo apt-get install git
```

---

### Step 1 — Get a MuAPI API Key

1. Go to [https://muapi.ai/dashboard](https://muapi.ai/dashboard)
2. Sign up or log in
3. Navigate to **API Keys** → **Generate Key**
4. Copy the key — you'll need it in Step 3

---

### Step 2 — Clone the Repository

```bash
git clone <YOUR_GITHUB_REPO_URL> muapi-video-marketing
cd muapi-video-marketing
```

---

### Step 3 — Run Setup

```bash
bash scripts/setup.sh
```

The script will:
1. Check Node.js and npm versions
2. Install MCP server dependencies
3. Build the MCP server (`mcp-server/dist/index.js`)
4. Prompt you for your MuAPI API key
5. Print the exact config snippet to paste into Claude

At the end you'll see a JSON block like:

```json
{
  "mcpServers": {
    "muapi-video-marketing": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-server/dist/index.js"],
      "env": {
        "MUAPI_API_KEY": "your_key_here"
      }
    }
  }
}
```

Copy this — you'll use it in the next step.

---

### Step 4 — Add MCP Server to Claude

Choose the Claude client you're using:

#### Claude Code CLI

Open (or create) `~/.claude/settings.json` and add the `mcpServers` block:

```json
{
  "mcpServers": {
    "muapi-video-marketing": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-server/dist/index.js"],
      "env": {
        "MUAPI_API_KEY": "your_key_here"
      }
    }
  }
}
```

If `mcpServers` already exists, merge the `"muapi-video-marketing"` entry into it.

#### Claude Desktop (macOS)

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "muapi-video-marketing": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-server/dist/index.js"],
      "env": {
        "MUAPI_API_KEY": "your_key_here"
      }
    }
  }
}
```

Then **restart Claude Desktop** (Cmd+Q, reopen).

#### Claude Desktop (Windows)

Edit `%APPDATA%\Claude\claude_desktop_config.json` with the same JSON structure above.

---

### Step 5 — Install the Skill

Copy `SKILL.md` into your Claude skills directory so Claude knows how to use the tools:

```bash
# Claude Code — global (available in all projects)
cp SKILL.md ~/.claude/skills/muapi-video-marketing.md

# Claude Code — project-only
mkdir -p .claude/skills
cp SKILL.md .claude/skills/muapi-video-marketing.md
```

For Claude Desktop, place it in your skills folder if you have one configured.

---

### Step 6 — Verify It Works

**Test the MCP server directly** (optional sanity check):

```bash
MUAPI_API_KEY=your_key node mcp-server/dist/index.js
# Should print: [muapi-mcp] Server started on stdio. Tools: muapi_generate_image, ...
# Press Ctrl+C to exit
```

**Test with Claude:**

Open Claude Code CLI in any project and ask:

```
Create a single test image: a sleek dark background with lime-green neon text "Hello World", photorealistic, 8k
```

Claude will call `muapi_generate_image` and return an image URL. If you get a URL back — you're set up correctly.

## MCP Tools Reference

| Tool | Description |
|------|-------------|
| `muapi_generate_image` | Text → image via Flux Dev (12B params) |
| `muapi_image_to_video` | Static image → animated video clip |
| `muapi_generate_video` | Text → video (no source image) |
| `muapi_create_music` | Generate background music via Suno V5 |
| `muapi_lipsync` | Sync lips to audio track |
| `muapi_video_effects` | Apply Wan AI cinematic effects |
| `muapi_upload_file` | Upload local file → CDN URL |
| `muapi_poll_result` | Check async task status |
| `muapi_run_workflow` | Execute a saved MuAPI workflow |
| `muapi_list_workflows` | Discover available workflows |

## Pipeline Deep Dive

### Standard Marketing Video

```
1. Claude generates storyboard (3–8 scenes as JSON)
2. muapi_generate_image × N   → scene images (Flux Dev, parallel)
3. muapi_image_to_video × N   → scene clips (Kling Pro, sequential)
4. muapi_create_music × 1     → background track (Suno V5)
5. Claude delivers: clip URLs + music URL + assembly guide
```

### Asset-First (User provides images/video)

```
1. muapi_upload_file           → CDN URL for each local asset
2. muapi_image_to_video        → animate provided images
3. muapi_create_music          → background track
```

### Spokesperson Video

```
1. muapi_generate_image        → scene with person
2. muapi_image_to_video        → animate
3. User provides voiceover
4. muapi_lipsync               → sync lips to voiceover
```

## Model Guide

### Image Models

| Model ID | Quality | Speed | Best For |
|----------|---------|-------|----------|
| `flux-dev` | ★★★★★ | Medium | Default, high quality |
| `flux-schnell` | ★★★★ | Fast | Drafts, prototyping |
| `midjourney-v7` | ★★★★★ | Slow | Premium aesthetics |

### Video Models

| Model ID | Quality | Speed | Best For |
|----------|---------|-------|----------|
| `kling-pro` | ★★★★★ | Slow | Product shots, realism |
| `kling-standard` | ★★★★ | Medium | General use |
| `minimax-pro` | ★★★★★ | Medium | Dynamic motion, social |
| `runway-gen3` | ★★★★ | Fast | Artistic, creative |
| `wan-2.1` | ★★★★★ | Slow | High fidelity, speech |

### Music (Suno — `muapi_create_music`)

Valid model values: `"V5"` (default), `"V4_5"`, `"V4"`, `"V3_5"`

The `style` parameter is **required** by the API — always pass concise style tags:

```
style: "edm, electronic, no vocals"
style: "orchestral, cinematic, no lyrics"
style: "lo-fi, chill, instrumental"
```

Prompt tips:
- `no vocals` or `instrumental` — critical for background music
- Match BPM energy to content pace
- Examples: `"upbeat corporate, 120bpm, no vocals"` | `"cinematic orchestral, epic build, no lyrics"`

## Assembly

After Claude delivers clip and music URLs, use any video editor to assemble:

- **CapCut** (free, mobile/desktop) — import clips, arrange timeline, add music
- **DaVinci Resolve** (free, desktop) — professional grade
- **FFmpeg** (command line) — see below

```bash
# 1. Create clip list
cat > clips.txt << 'EOF'
file 'scene1.mp4'
file 'scene2.mp4'
file 'scene3.mp4'
EOF

# 2. Merge clips
ffmpeg -f concat -safe 0 -i clips.txt -c copy merged.mp4

# 3. Add music (mix at lower volume)
ffmpeg -i merged.mp4 -i music.mp3 \
  -filter_complex "[1:a]volume=0.3[music];[0:a][music]amix=inputs=2:duration=first" \
  -c:v copy final.mp4
```

## Troubleshooting

**Server not connecting**
```bash
# Test manually
MUAPI_API_KEY=your_key node mcp-server/dist/index.js
```

**401 Unauthorized**
- Check `MUAPI_API_KEY` is set correctly in the MCP config

**404 Not Found on video/workflow endpoints**
- Some MuAPI endpoints change over time. Image generation and video effects are stable.
- Check [https://muapi.ai/docs](https://muapi.ai/docs) for current endpoint paths
- For `muapi_generate_video` / `muapi_image_to_video` failures, try `muapi_video_effects` as an alternative to animate images

**422 on music generation**
- Ensure you pass a `style` field (e.g. `"electronic, no vocals"`)
- Use a valid model: `"V5"`, `"V4_5"`, `"V4"`, or `"V3_5"`

**Task failed / timeout**
- Check MuAPI status at [https://muapi.ai/dashboard](https://muapi.ai/dashboard)
- Retry with a simpler/shorter prompt
- Use `wait_for_result: false` and poll manually with `muapi_poll_result`

**NSFW content flagged**
- Remove suggestive language from prompts
- Use explicit style descriptors: `"professional product photography"`, `"clean corporate"`

**Build errors after clone**
```bash
cd mcp-server
rm -rf node_modules dist
npm install
npm run build
```
